# UML (Unified Modeling Language)
OMG에서 표준으로 채택한 통합 모델링 언어  
모델을 만드는 표준언어  
(모델: 어떤 것을 실제로 만들 때 미리 검증하는 것)

## UML의 사용 목적
- 협업 시 의사 소통, 설계 논의
- 전체 시스템 구조 및 클래스 의존성 파악
- 유지보수를 위한 설계 문서

# 클래스 다이어그램
UML은 구조 다이어그램(7개), 행위 다이어그램(7개)로 구성
- 구조 다이어그램: 시스템의 개념, 관계 등의 측면에서 요소를 나타냄
- 행위 다이어그램: 각 요소/ 요소 간의 변화나 흐름, 주고받는 데이터 등의 동작을 보기 위한 것
- 클래스 다이어그램: 구조 다이어그램에 해당
  - 클래스 내부의 정적 내용이나 클래스 사이의 관계를 표시하는 다이어그램
  - 시스템 일부/ 전체 구조를 나타냄
  - 명확한 의존관계

## 목적에 따른 UML
1. 개념
- 문제 도메인의 구조를 나타내며 개념과 추장적 개념을 기술하기 위함
- 의미론적 규칙에 얽매이지 않아서 의미나 해석이 모호함

2. 명세 및 구현
- 소프트웨어의 설계/완성된 소프트웨어의 구현 설명 목적 등으로 사용
- 소스코드를 변경하거나 구현된 소스코드를 설명하려고 사용함

## 클래스 다이어그램의 요소
1. Class
   - 보통 3개의 구획으로 나누어 클래스의 이름, 속성, 기능을 표기 (속성, 기능 생략가능, 이름 필수)
   - 클래스의 세부사항은 필드와 메서드의 Access modifier(접근제한자), 필드명(메서드명), 데이터타입, parameter(매개변수), 리턴 타입 등으로 나타냄

2. 스테레오 타입
- UML에서 제공하는 기본 요소 외에 추가 확장요소를 나타냄
- «과 » 기호를 길러멧(guillemet)이라고 하며 길러멧사이에 적는다
- 스테레오 타입으로 많이 사용하는 것은 «interface», «utility», «abstract», «enumeration»,
